# Issue #7 — Legacy LD-Signature cryptosuites (`DataProofsDotnet.Legacy`)

GA 1.0.0. Ship `Ed25519Signature2020` + `EcdsaSecp256r1Signature2019` as opt-in
`ICryptosuite`s (JCS + RDFC variants) that create/verify through `DataIntegrityProofPipeline`,
byte-identical to zcap-dotnet.

## Plan (approved)

- [x] New `DataProofsDotnet.Legacy` package (refs Core + Rdfc); not in `CreateDefault`.
- [x] `LegacyCryptosuiteBase` + two sealed suites + `LegacyCanonicalization` enum + internal
      `EcdsaSignatureEncoding` (DER→P1363).
- [x] JCS variant = `JCS(document + nested proof-minus-proofValue-minus-cryptosuite)` fed
      directly to the signer; RDFC variant = `SHA-256(RDFC(proofOptions)) ‖ SHA-256(RDFC(doc))`.
- [x] Emits `type:"<suite>"`, no `cryptosuite`, base58-btc `proofValue`. No Core changes.
- [x] Test project + cross-stack zcap golden vector; full-solution build/test.
- [x] Adversarial review workflow (spec → implement → tests → review/verify).

## Adversarial findings & resolutions

- [x] **#0/#3 (HIGH) JCS clobbered pipeline-injected previousProof chain → fail-open.**
      JCS now **fails closed** when the unsecured document already carries a `proof` member
      (create throws; verify treats JCS as inapplicable so the RDFC fallback runs, net
      fail-closed). RDFC binds chains natively (predecessors live in the canonicalized doc).
      Tests: `LegacyChainBindingTests`.
- [x] **#1 (MEDIUM) JCS didn't strip null object members → byte divergence from zcap.**
      Added `StripNullMembers` to the JCS path (mirrors zcap; keeps null array elements).
      Tests: `JcsNullMemberByteIdentityTests`.
- [x] **#4/#5 (CRIT/HIGH) RDFC drops `@context`-undefined proof members; tests JCS-only.**
      Inherent JSON-LD/RDFC behavior (shared by conformant `rdfc-*`). Documented on the suite,
      enum, CHANGELOG, PRD; added honest characterization tests (`RdfcExtensionBinding…`) that
      pin JCS-binds vs RDFC-drops so there is no false assurance.
- [x] **#2 (LOW) P-256 high-S malleability accepted.** Matches zcap + modern suites; enforcing
      low-`s` would break byte-compat. Documented (proofValue ≠ unique id).
- [x] **#6/#7/#8 (LOW) scratch probe files.** Never committed (worktree artifacts only); tree
      verified clean.
